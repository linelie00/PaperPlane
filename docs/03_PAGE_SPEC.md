# 03_PAGE_SPEC.md

# PaperPlane 페이지 명세

이 문서는 PaperPlane 초기 MVP의 각 페이지 목적과 필수 UI 요소를 정의한다.
모든 화면은 [08_UI_DESIGN_GUIDE.md](08_UI_DESIGN_GUIDE.md)를 기준으로 한다.
기본 배경은 흰색 + 옅은 하늘색 그라데이션, 포인트 컬러는 청록색(`#22C7C7`)과 하늘색(`#38BDF8`)을 사용한다.

## 페이지 목록

| 경로 | 페이지 | 인증 |
|------|--------|------|
| `/` | 랜딩 페이지 | 불필요 |
| `/login` | 로그인 페이지 | 불필요 |
| `/signup` | 회원가입 페이지 | 불필요 |
| `/dashboard` | 창작자 대시보드 | 필요 |
| `/works` | 작품 목록 페이지 | 필요 |
| `/works/new` | 작품 업로드 페이지 | 필요 |
| `/works/[workId]` | 작품 상세 페이지 | 필요 (소유자) |
| `/read/[publicSlug]` | 공개 독자 뷰어 | 불필요 |

---

## 1. 랜딩 페이지 `/`

### 목적

서비스를 소개하고 창작자에게 가치를 전달하여 회원가입/업로드로 유도한다.

### 필수 UI 요소

* **Header**: PaperPlane 로고(`✈ PaperPlane`), 로그인 링크, 시작하기 버튼
* **Hero Section** — 반드시 아래 캐치프레이즈를 포함한다.

  > 언어가 달라져도,
  > 이야기가 주는 설렘은 같아야 한다

  * 서브 카피: "PaperPlane은 창작자의 이야기를 AI로 번역하고, 해외 독자가 바로 읽을 수 있는 뷰어와 반응 확인 기능을 제공합니다."
  * Primary CTA: "지금 작품 업로드하기"
  * Secondary CTA: "샘플 작품 보기"
  * 배경: 옅은 하늘색 그라데이션 + 점선 비행 경로 장식
* **Problem Section**: 창작자의 문제 카드 (번역 비용 부담 / 해외 독자 반응 확인 어려움 / 작품 발견 경로 부족)
* **Solution Section**: PaperPlane 해결책 카드 (작품 업로드 / AI 자동 번역 / 공개 뷰어 생성 / 독자 유입 확인 / 댓글 반응 수집)
* **How It Works Section**: 3단계 (1. 작품 업로드 → 2. AI 번역 생성 → 3. 공개 링크로 반응 확인)
* **Creator Dashboard Preview**: 전체 조회수 / 오늘 조회수 / 댓글 수 / 주요 유입 경로 / 최근 댓글 미리보기 카드
* **Reader Viewer Preview**: 작품 제목 / 번역 본문 / 댓글 영역 (모바일 뷰어 느낌)
* **CTA Section**: 감성 문구("당신의 이야기를 더 먼 곳의 독자에게 보내보세요.") + "첫 작품 업로드하기" 버튼

### 주의사항

* CTA 버튼은 3개 이상 나열하지 않는다.
* 종이비행기 일러스트를 과하게 반복하지 않는다.

## 2. 로그인 페이지 `/login`

### 목적

기존 창작자가 이메일과 비밀번호로 로그인한다.

### 필수 UI 요소

* 이메일 입력 필드
* 비밀번호 입력 필드
* 로그인 버튼 (Primary)
* 회원가입 이동 링크 ("아직 계정이 없으신가요? 회원가입")
* 로그인 실패 시 에러 메시지 영역

### 동작

* 로그인 성공 → `/dashboard` 이동
* 로그인 실패 → 이메일 또는 비밀번호 오류 메시지 표시

---

## 3. 회원가입 페이지 `/signup`

### 목적

창작자 계정을 새로 생성한다.

### 필수 UI 요소

* 닉네임 입력 필드
* 이메일 입력 필드
* 비밀번호 입력 필드
* 비밀번호 확인 입력 필드
* 회원가입 버튼 (Primary)
* 로그인 이동 링크 ("이미 계정이 있으신가요? 로그인")
* 유효성 에러 메시지 영역

### 동작 / 유효성

* 이메일 형식 검증
* 비밀번호와 비밀번호 확인 일치 검증
* 이메일 중복 시 에러 표시
* 가입 성공 → 로그인 페이지로 이동

---

## 4. 창작자 대시보드 `/dashboard`

### 목적

창작자가 자신의 작품과 독자 반응을 한눈에 확인한다.
복잡한 분석 툴처럼 보이지 않게 하고, 가장 궁금한 정보를 우선 보여준다.

### 필수 UI 요소

* **상단 요약 카드**
  * 전체 작품 수
  * 전체 조회수
  * 오늘 조회수
  * 전체 댓글 수
* **최근 업로드 작품** 리스트
* **최근 댓글** 리스트
* **작품별 조회수 요약** (작품 제목 / 공개 여부 / 번역 상태 뱃지 / 조회수 / 댓글 수 / 공개 링크 복사 버튼)
* **주요 유입 경로** (referrer / utm source 상위 목록)

### 번역 상태 뱃지

| 상태 | 라벨 | 색상 |
|------|------|------|
| `pending` | 대기 중 | 회색 |
| `processing` | 번역 중 | 하늘색 |
| `completed` | 번역 완료 | 청록색 |
| `failed` | 번역 실패 | 빨간색 |

### 반응형

* 지표 카드는 모바일에서 1열 또는 2열로 정리한다.

---

## 5. 작품 목록 페이지 `/works`

### 목적

창작자가 업로드한 작품 목록을 확인하고 상세로 이동한다.

### 필수 UI 요소 (작품 카드/행 단위)

* 작품 제목
* 원문 언어 / 번역 언어
* 공개 여부
* 조회수
* 댓글 수
* 번역 상태 뱃지
* 생성일
* 상세 페이지 이동 링크
* "새 작품 업로드" 버튼 → `/works/new`

### 빈 상태

* 업로드한 작품이 없을 때 "첫 작품을 업로드해 보세요" 안내와 업로드 버튼 표시

---

## 6. 작품 업로드 페이지 `/works/new`

### 목적

창작자가 작품을 업로드한다. 초기 MVP에서는 파일 업로드가 아닌 텍스트 입력 방식을 우선한다.
단계형 폼으로 구성하고, 긴 텍스트를 붙여넣어도 답답하지 않게 넓게 배치한다.

### 권장 단계

```txt
1. 작품 정보 입력
2. 원문 텍스트 입력
3. 번역 언어 선택
4. 번역 시작
```

### 필수 입력값

* 작품 제목 (필수)
* 작품 소개
* 장르
* 태그 (복수)
* 원문 언어 (기본: 한국어)
* 번역 대상 언어 (기본: 영어)
* 원문 텍스트 (필수, 넓은 textarea)

### 동작

* "번역 시작하기" 클릭 → 작품 저장 → 번역 요청 → 상세 페이지로 이동
* 작품 제목 또는 원문 텍스트가 비면 저장하지 않고 에러 표시
* 너무 긴 텍스트는 제한 메시지 표시

---

## 7. 작품 상세 페이지 `/works/[workId]`

### 목적

창작자가 업로드한 작품과 번역 결과를 확인하고 공개를 관리한다.
소유자만 접근할 수 있다.

### 필수 UI 요소

* 작품 제목
* 작품 소개 / 장르 / 태그
* 원문 텍스트
* 번역 텍스트
* 번역 상태 표시 (상태별 UI)
* 공개 여부 토글
* 공개 링크 복사 버튼 (Secondary)
* 조회수
* 댓글 목록
* "번역 다시 생성" 버튼 (`failed` 시 강조)

### 상태별 UI

* `pending`: "번역 대기 중"
* `processing`: "번역 중" 로딩 표시
* `completed`: 번역문 표시 + 공개 토글 활성화
* `failed`: "번역 실패" + 다시 시도 버튼

### 권한

* 작품 소유자가 아니면 접근 불가 처리

---

## 8. 공개 독자 뷰어 페이지 `/read/[publicSlug]`

### 목적

독자가 번역된 작품을 읽는 공개 페이지이다. 로그인이 필요 없다.
가장 간결하고 가독성이 좋아야 하며, 모바일 대응이 필수이다.

### 필수 UI 요소

* 작품 제목
* 작품 소개
* 번역 언어 표시
* 창작자 이름
* 번역된 본문 (가독성 최우선)
* 댓글 목록
* 댓글 작성 폼 (닉네임 + 댓글 내용 + 작성 버튼)

### 본문 영역 권장 스타일

```css
max-width: 720px;
margin: 0 auto;
font-size: 18px;
line-height: 1.9;
color: #102A43;
```

### 접근 시 처리

```txt
1. 공개 작품인지 확인한다.
2. 조회수를 1 증가시킨다.
3. referrer, utm_source, utm_medium, utm_campaign, userAgent, ipHash 등 유입 정보를 저장한다.
4. 번역된 본문을 보여준다.
```

### 예외 처리

* 존재하지 않는 publicSlug → 404 페이지
* 비공개 작품 → 접근 불가 메시지

### 댓글 규칙

* 닉네임 필수, 댓글 내용 필수
* 댓글 길이 제한 500자
* XSS 방지를 위해 HTML 태그 제거(sanitize)
* 광고, 추천 등은 제외
