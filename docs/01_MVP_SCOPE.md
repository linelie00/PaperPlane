# PaperPlane MVP 범위 정의

이 문서는 PaperPlane 초기 MVP의 범위를 정의한다. 모든 개발은 이 문서의 범위를 기준으로 한다.
판단 기준은 항상 "업로드 후 바로 번역본을 공개하고 독자 반응을 확인하는 흐름"의 완성이다.

## 1. 프로젝트 목표

PaperPlane은 롱테일 창작자를 위한 AI 글로벌 창작물 현지화 서비스이다.

초기 MVP의 목표는 다음과 같다.

* 창작자가 웹소설 또는 웹툰 대사 텍스트를 업로드하면 AI가 자동으로 번역한다.
* 독자가 번역된 작품을 웹 뷰어에서 읽을 수 있다.
* 창작자가 독자 유입 수와 댓글 반응을 확인하여 해외 독자 반응을 빠르게 검증할 수 있다.

핵심 메시지는 다음 캐치프레이즈로 표현한다.

> 언어가 달라져도, 이야기가 주는 설렘은 같아야 한다

초기 MVP는 복잡한 현지화 전체 파이프라인이 아니라, 전체 사용 흐름의 완성을 우선한다.
번역 품질 고도화보다 "업로드 → 번역 → 공개 → 반응 확인" 흐름이 끊김 없이 동작하는 것이 더 중요하다.

## 2. MVP 포함 기능

초기 MVP는 다음 11개 핵심 기능으로 범위를 제한한다.

| # | 기능 | 설명 |
|---|------|------|
| 1 | 회원가입 | 이메일/비밀번호 기반 창작자 계정 생성 |
| 2 | 로그인 | 이메일/비밀번호 기반 인증 |
| 3 | 로그아웃 | 세션/토큰 만료 처리 |
| 4 | 작품 업로드 | 텍스트 입력 방식으로 작품 메타데이터와 원문 저장 |
| 5 | AI 자동 번역 | 서버에서 LLM API 호출, 한국어 → 영어 번역 |
| 6 | 작품 목록 | 로그인 창작자의 작품 목록 조회 |
| 7 | 작품 상세 | 원문/번역문 확인, 공개 토글, 공개 링크 복사 |
| 8 | 공개 독자 뷰어 | 로그인 없이 접근 가능한 번역본 읽기 페이지 |
| 9 | 조회수 기록 | 뷰어 접근 시 조회수 및 유입 정보 저장 |
| 10 | 댓글 작성 및 조회 | 로그인 없이 닉네임 기반 댓글 작성/조회 |
| 11 | 창작자 대시보드 | 전체 조회수, 댓글 수, 작품별 요약, 유입 경로 확인 |

### 보조 기능 (포함)

위 핵심 기능을 지원하기 위해 다음을 함께 구현한다.

* 로그인 사용자 보호 라우팅
* 공개 링크(publicSlug) 생성
* 유입 경로(referrer, utm) 기록
* 번역 상태 관리 (`pending` / `processing` / `completed` / `failed`)
* 번역 실패 시 다시 시도 기능

## 3. MVP 제외 기능

다음 기능은 초기 MVP에서 구현하지 않는다.
단, 이후 확장을 고려하여 DB 구조와 API는 확장 가능하게 작성한다.

* 웹툰 이미지 OCR
* 식자 자동화
* 결제
* 후원
* 추천 알고리즘
* 관리자 페이지
* 다국어 동시 번역
* 정교한 용어집 자동 생성
* 캐릭터 말투 유지 고도화
* 실시간 알림

추가로 다음도 MVP 이후로 미룬다.

* 번역 품질 점수화
* 회차별 작품 관리
* 독자 로그인 / 독자 계정
* 댓글 삭제 및 욕설 필터링
* 복잡한 권한 관리

## 4. 완료 기준

초기 MVP는 다음 흐름이 처음부터 끝까지 정상 동작하면 완료로 본다.

```txt
1. 창작자가 회원가입한다.
2. 창작자가 로그인한다.
3. 창작자가 작품 텍스트를 업로드한다.
4. AI가 작품을 영어로 자동 번역한다.
5. 창작자가 번역 결과를 확인한다.
6. 창작자가 작품을 공개한다.
7. 독자가 공개 링크로 접속한다.
8. 독자가 번역 작품을 읽는다.
9. 독자가 댓글을 남긴다.
10. 창작자가 조회수와 댓글을 확인한다.
```

### 세부 완료 조건

* 비공개 작품은 공개 링크로 접근할 수 없다.
* 존재하지 않는 publicSlug 접근 시 404를 보여준다.
* 번역 실패 시에도 작품 정보와 원문은 유지되고 다시 시도할 수 있다.
* 창작자는 자신의 작품만 수정할 수 있다.
* LLM API Key는 서버에서만 사용되며 프론트엔드에 노출되지 않는다.

## 5. 개발 우선순위

다음 순서로 단계적으로 개발한다. 앞 단계가 동작해야 다음 단계로 진행한다.

### 1단계: 기본 인증

* 회원가입
* 로그인
* 로그아웃
* 로그인 사용자 보호 라우팅

### 2단계: 작품 업로드

* 작품 업로드 폼
* 작품 저장
* 작품 목록
* 작품 상세

### 3단계: AI 번역

* 서버에서 LLM API 호출
* 번역 결과 저장
* 번역 상태 표시
* 번역 실패 처리

### 4단계: 독자 뷰어

* 공개 링크 생성
* 공개 작품 조회
* 번역문 뷰어
* 모바일 가독성 개선

### 5단계: 댓글

* 댓글 작성
* 댓글 조회
* 작품 상세에서 댓글 확인

### 6단계: 유입 확인

* 조회수 저장
* referrer / utm 저장
* 작품별 조회수 표시
* 대시보드 구현

## 6. 개발 시 주의사항

* MVP 범위를 벗어나는 기능을 먼저 구현하지 않는다.
* 번역 품질 고도화보다 전체 사용 흐름 완성을 우선한다.
* API Key는 절대 프론트엔드에 노출하지 않는다.
* publicSlug는 예측하기 어렵게 생성한다.
* 창작자가 자신의 작품만 수정할 수 있도록 권한 체크를 반드시 한다.
* 공개 뷰어는 로그인 없이 접근 가능해야 한다.
* 비밀번호는 반드시 해시 처리하고, IP는 원문 저장하지 않고 해시 처리한다.
