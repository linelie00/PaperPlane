# PaperPlane 사용자 흐름

이 문서는 PaperPlane 초기 MVP의 사용자 흐름을 정의한다.
PaperPlane에는 두 종류의 사용자가 있다.

* 창작자: 작품을 업로드하고 번역하여 공개하고 반응을 확인한다. 로그인이 필요하다.
* 독자: 공개된 번역 작품을 읽고 댓글을 남긴다. 로그인이 필요 없다.

## 1. 창작자 사용자 흐름

창작자의 전체 흐름은 다음과 같다.

```txt
회원가입
→ 로그인
→ 작품 업로드
→ AI 자동 번역
→ 번역 결과 확인
→ 작품 공개
→ 공개 링크 공유
→ 대시보드에서 유입 및 댓글 확인
```

### 1.1 회원가입 / 로그인

```txt
랜딩 페이지 진입
→ "시작하기" 또는 "지금 작품 업로드하기" 클릭
→ (미로그인 시) 회원가입 페이지로 이동
→ 닉네임, 이메일, 비밀번호, 비밀번호 확인 입력
→ 회원가입 완료
→ 로그인 페이지로 이동
→ 이메일/비밀번호 로그인
→ 대시보드로 이동
```

* 비밀번호는 서버에서 해시 처리하여 저장한다.
* 로그인 성공 시 세션 또는 JWT 토큰을 발급한다.
* 보호 라우트(`/dashboard`, `/works`, `/works/new`, `/works/[workId]`)는 미로그인 시 로그인 페이지로 리다이렉트한다.

### 1.2 로그아웃

```txt
헤더의 로그아웃 클릭
→ 세션/토큰 만료
→ 랜딩 또는 로그인 페이지로 이동
```

## 2. 독자 사용자 흐름

독자의 전체 흐름은 다음과 같다. 로그인은 필요하지 않다.

```txt
공개 링크 접속
→ 번역 작품 읽기
→ 댓글 작성
```

### 2.1 상세 흐름

```txt
창작자가 공유한 공개 링크 /read/[publicSlug] 접속
→ 서버가 공개 작품인지 확인
→ 공개 작품이면 조회수 1 증가 및 유입 정보 저장
→ 번역된 본문, 작품 소개, 창작자 이름 표시
→ 댓글 목록 확인
→ 닉네임과 댓글 내용 입력 후 댓글 작성
→ 작성된 댓글이 목록에 표시
```

### 2.2 접근 예외 처리

* 존재하지 않는 publicSlug → 404 페이지
* 비공개 작품 → 접근 불가 메시지 표시
* 번역이 완료되지 않은 작품 → 공개되지 않도록 처리(공개 토글은 번역 완료 후 권장)

## 3. 작품 업로드 후 번역 / 공개 흐름

작품 업로드부터 공개까지의 상세 흐름이다.

```txt
작품 업로드 페이지 /works/new 진입
→ 작품 정보 입력 (제목, 소개, 장르, 태그)
→ 원문 텍스트 입력
→ 원문 언어 / 번역 대상 언어 선택 (기본: 한국어 → 영어)
→ "번역 시작하기" 클릭
```

서버 처리 순서:

```txt
1. 로그인 사용자 확인
2. 작품 메타데이터 저장 (Work)
3. 원문 텍스트 저장 (WorkContent.originalText)
4. 번역 상태를 processing 으로 변경
5. 서버에서 LLM API 호출하여 번역 실행
6. 성공 시 translatedText 저장 및 상태 completed
7. 실패 시 상태 failed (원문은 유지)
8. 작품 상세 페이지 /works/[workId] 로 이동
```

### 3.1 번역 상태별 처리

| 상태 | 의미 | UI |
|------|------|----|
| `pending` | 번역 대기 중 | "번역 대기 중" 표시 |
| `processing` | 번역 진행 중 | "번역 중" 로딩 표시 |
| `completed` | 번역 완료 | 번역문 표시, 공개 가능 |
| `failed` | 번역 실패 | "번역 실패" + 다시 시도 버튼 |

### 3.2 공개 흐름

```txt
작품 상세 페이지에서 번역 완료 확인
→ 공개 여부 토글을 ON 으로 변경
→ publicSlug 가 없으면 생성
→ 공개 링크 복사 버튼으로 링크 복사
→ 외부(SNS 등)에 링크 공유
```

* publicSlug는 예측하기 어렵게 생성한다.
* 비공개로 전환하면 공개 링크 접근이 차단된다.

### 3.3 다시 번역 흐름

```txt
번역 실패 또는 재번역이 필요한 경우
→ "다시 번역하기" 클릭
→ POST /api/translate (workId)
→ 번역 상태 processing
→ 번역 재실행
→ 성공 시 completed, 실패 시 failed
```

## 4. 독자 유입 및 댓글 확인 흐름

창작자가 해외 독자 반응을 확인하는 흐름이다.

```txt
독자가 공개 뷰어 접속
→ 조회수 1 증가
→ referrer, utm_source, utm_medium, utm_campaign, userAgent, ipHash 저장 (ViewLog)
→ 독자가 댓글 작성 (Comment)
↓
창작자가 대시보드 /dashboard 접속
→ 전체 조회수 / 오늘 조회수 확인
→ 전체 댓글 수 확인
→ 작품별 조회수 / 댓글 수 확인
→ 주요 유입 경로(referrer / utm) 확인
→ 최근 댓글 확인
```

### 4.1 수집 지표

* 작품별 조회수
* 날짜별 조회수
* 유입 경로 referrer
* utm_source / utm_medium / utm_campaign
* 댓글 수

### 4.2 대시보드 표시 지표

* 전체 조회수
* 오늘 조회수
* 전체 댓글 수
* 공개 작품 수
* 작품별 조회수 / 댓글 수
* 주요 유입 경로
* 최근 댓글

### 4.3 개인정보 처리

* IP는 원문 저장하지 않고 해시(ipHash) 처리한다.
* 조회수 중복 방지는 초기 MVP에서 엄격하게 구현하지 않는다.
